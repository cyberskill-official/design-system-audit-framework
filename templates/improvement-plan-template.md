## Reading guide

Each phase below has:
- **Goal** — what unlocks at the end.
- **Audit lift** — estimated combined-score impact when completed.
- **Step list** — actions in execution order. Each step has a clear "done when" condition.
- **Dependencies** — what must land before this phase starts.
- **Risk** — what to watch for.

Phases run mostly sequentially but waves within a phase can overlap. The plan does **not** include calendar dates — execute at whatever pace your team supports.

## What this plan does NOT do

This plan deliberately does **not** chase criteria that gate on:

1. **Year-over-year trend data** — needs ≥ 2 annual audits with measurable deltas.
2. **External validation** — independent A11y audit, conference talk acceptance, sustained customer adoption.
3. **Calendar-bound milestones** — sales cycles, vendor delivery, regulatory review.

These criteria typically sit at score 4. They lift to 5 *naturally* as Phases execute and time passes. Pushing them artificially is theatre.

## How to use this plan

Each step has a "done when" condition. Track progress in a running list at `_history.md` under "Phase milestones". Append a row when each phase wave completes.

When ambiguity arises about what to do next, use this priority order:

1. **Anything `@Human[manual]` blocking the next audit's score.** These typically need the most calendar time.
2. **Anything that unlocks measurement.** Public deploy unlocks 4 categories at once — outsized leverage.
3. **Anything that compounds.** First customer case study → second customer pitch leverages it → third becomes self-sustaining.
4. **Doctrine improvements.** Token additions, new patterns, vertical packs. These move the score 1–2 pp at a time.

The framework's own quarterly DYNAMIC re-score and annual full audit provide the pacing. This plan provides the content for those cycles.
